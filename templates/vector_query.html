What does the ETH-UDK Vector Query do?

The ETH-UDK Vector Query is an AI-powered discovery tool that helps you find subject terms from the ETH-UDK classification that semantically match the content you provide. You can enter a title, abstract, and/or table of contents. Instead of performing a traditional keyword search, the system uses semantic vector search ‚Äì meaning it compares the underlying meaning of the text rather than exact words.

What happens behind the scenes?
  • Text input is analyzed: Your inputs are combined into one continuous text.
  • Embedding creation: The text is transformed into a vector using OpenAI's GPT-4 embedding model. This vector is a mathematical representation of the meaning of your input.
  • Semantic search: This vector is compared against vectors in a Pinecone database containing ETH-UDK classification terms.
  • Classification depth filter: You can limit which classification levels (e.g., 5.0 to 9.0) are included in the results.
  • Ranked results: You receive a sorted list of the most semantically similar classification terms with metadata like ID, level, UDC notation, variants, and broader terms.
Key Notes
  • The system is language-agnostic: it finds matches regardless of the language used in your input or the classification term.
  • Results are based on meaning, not keyword occurrence.
  • The AI has no prior knowledge of the ETH-UDK ‚Äì it simply compares meaning using mathematical similarity.
Experimental Tool

This tool represents an initial prototype for performing semantic queries on the ETH-UDK classification. It is intended as a foundation for the future development of more powerful, tailored, and automated workflows using agentic AI methods. Such workflows could integrate specific rules, constraints, and context-sensitive logic relevant to subject indexing ‚Äî enabling more accurate, scalable, and standards-compliant classification processes through Large Language Models and intelligent agents.
